# 프로젝트 정의서

## 1. 프로젝트 명칭

**WorkDrive** — 클라우드 파일 공유 시스템

## 2. 개발 시스템 명칭

**WorkDrive Cloud File Sharing System**  

## 3. 개발 시스템 설명

WorkDrive는 조직 구성원 간 효율적인 협업을 지원하기 위한 클라우드 기반 파일 공유 시스템이다.  
사용자는 파일을 업로드·다운로드하고, 팀원과 공유하며, 역할 기반으로 권한을 설정해 안전하게 협업할 수 있다.  
파일 버전 관리와 실시간 동기화를 통해 동시 작업 시 충돌과 혼선을 줄인다.  
웹 기반으로 제공하여 설치 없이 브라우저에서 언제 어디서나 업무 자료에 접근할 수 있도록 설계한다.

## 4. 핵심 기능

| 기능 | 설명 |
|------|------|
| **파일 업로드·다운로드 및 클라우드 저장** | 문서·이미지 등 다양한 파일을 중앙 저장소에 올리고 필요 시 내려받기 |
| **파일·폴더 공유** | 팀원 또는 지정 사용자에게 항목 단위로 공유 |
| **파일 댓글** | 공유된 파일에 협업 의견을 남기며, 열람·수정·댓글 등 권한에 따라 작성 가능 여부를 구분 |
| **접근 권한 관리 (RBAC)** | 읽기 / 수정 / 댓글 / 관리자 등 역할에 따른 접근 제어 |
| **파일 버전 관리** | 변경 이력 보관 및 이전 버전으로 복구 |
| **실시간 동기화** | 서버에 저장된 내용으로, 업로드·수정·공유가 다른 사용자 화면에 **지체 없이 반영**되는 것을 말한다. |
| **검색** | 파일명, 사용자, 날짜 등 조건으로 빠르게 찾기 |

### 실시간 동기화의 의미

“실시간 동기화”가 **무엇을 의미하는지**는 제품마다 다르므로, WorkDrive에서의 범위를 다음과 같이 구분해 둔다.

| 비교 대상 | 의미 | WorkDrive에서의 위치 |
|-----------|------|----------------------|
| **Google Docs 유형** | 한 문서를 여러 사람이 **동시에 편집**하고, 커서·문자 단위로 즉시 반영되는 **실시간 공동 편집** | 본 프로젝트의 **핵심 범위에 포함하지 않는다.** 필요 시 [품질 문서](Quality_Attribute_Measurement.md)에서 언급한 것처럼 향후 확장 로드맵으로 검토할 수 있다. |
| **Dropbox·OneDrive 유형** | 로컬 폴더와 클라우드 간 **파일 단위 동기화**(클라이언트가 백그라운드에서 올리고 받아 최신 유지) | 웹 브라우저 중심이면 **“목록·메타데이터·다운로드할 파일 버전이 서버와 일치”**하는 경험에 가깝다. 별도 데스크톱 동기화 클라이언트를 두면 같은 원리로 설명할 수 있다. |
| **WorkDrive(본 문서)** | **서버에 반영된 파일·버전·공유 상태**가 협업자에게 **새로고침 없이 또는 짧은 지연으로** 보이도록 하고, 동시에 같은 파일을 덮어쓸 때는 **버전 관리·충돌 안내**로 혼선을 줄인다. | 위 Dropbox 유형과 같이 **“클라우드가 최신이며 UI가 그에 맞춰 갱신”**에 초점을 둔다. **문서 안에서 동시 타이핑**까지 보장하는 것은 아니다. |

정리하면, **구글 닥스 스타일의 실시간 공동 편집이 아니라**, **클라우드 저장소와 화면 상태의 일치(및 버전·권한에 따른 최신 반영)**를 말한다.

## 5. 기존 유사 소프트웨어 조사 및 분석

### 1. Google Drive
- **장점:** 실시간 협업(문서 공동 편집 등)과 검색이 강함  
- **단점:** 기업·조직 단위의 세분화된 권한·정책 맞춤에는 한계가 있을 수 있음  

### 2. Dropbox
- **장점:** 직관적인 UI와 안정적인 동기화  
- **단점:** 파일 공유·협업 워크플로는 상대적으로 단순한 편  

### 3. Microsoft OneDrive
- **장점:** Microsoft 365·Office와 연동이 뛰어남  
- **단점:** 비 MS 환경·타 플랫폼과의 연동은 제약이 생길 수 있음  

### WorkDrive의 차별성
- **조직 중심 권한:** 부서·프로젝트·역할 단위로 세분화한 RBAC에 초점  
- **투명한 협업:** 활동·감사 로그로 누가 언제 무엇을 했는지 추적  
- **접근성:** 필수 기능 위주의 웹 UI로 빠른 탐색·업무 흐름 유지  
- **팀 협업 구조:** 팀/공유 스페이스 개념으로 폴더·권한을 일관되게 운영
